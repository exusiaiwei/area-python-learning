11  标准库

Python标准库是每个Python安装的一部分，它包含了许多有用的模块，包括操作文件、字符串处理、网络通信、日期和时间处理、数学、科学、加密等等。

熟悉Python标准库的模块，可以帮助我们更快速的编写程序，提高开发效率。

让我们探索一些常用的标准库模块。

11.1 sys模块

sys模块提供了一些变量和函数，用于与Python解释器进行交互。

11.1.1 sys.argv

正如我们在 Section 5.1.1 看到的，sys.argv是一个包含命令行参数的列表。

我们在此总结一下：

  • sys.argv[0]是脚本的名称
  • sys.argv[1]是第一个参数
  • sys.argv[2]是第二个参数
  • sys.argv的长度是命令行参数的个数

下面是一个简单的例子：

import sys

print(sys.argv)
['c:\\Conda_data\\envs\\main\\Lib\\site-packages\\ipykernel_launcher.py', '-f', 'C:\\Windows\\Temp\\tmp79v95aph.json', '--HistoryManager.hist_file=:memory:']

11.1.2 sys.version

sys.version包含了Python解释器的版本信息。

import sys

print(sys.version)
3.11.8 | packaged by conda-forge | (main, Feb 16 2024, 20:40:50) [MSC v.1937 64 bit (AMD64)]

sys模块有一个名为version_info的元组，它包含了Python解释器的版本信息。

11.1.3 sys.path

sys.path是一个包含了模块搜索路径的列表。

import sys

print(sys.path)
['c:\\Conda_data\\envs\\main\\python311.zip', 'c:\\Conda_data\\envs\\main\\DLLs', 'c:\\Conda_data\\envs\\main\\Lib', 'c:\\Conda_data\\envs\\main', '', 'c:\\Conda_data\\envs\\main\\Lib\\site-packages', 'c:\\Conda_data\\envs\\main\\Lib\\site-packages\\win32', 'c:\\Conda_data\\envs\\main\\Lib\\site-packages\\win32\\lib', 'c:\\Conda_data\\envs\\main\\Lib\\site-packages\\Pythonwin']

11.2 os模块

os模块提供了许多与操作系统交互的函数。

常见的操作包括：获取当前工作目录、更改目录、创建目录、删除目录、删除文件等。

11.2.1 os.getcwd()

os.getcwd()返回当前工作目录。

import os

print(os.getcwd())
C:\Code\MyProjects\My-Python-Learning

11.2.2 os.listdir()

os.listdir()返回指定目录下的文件和目录列表。

import os

print(os.listdir())
['.git', '.github', '.gitignore', '.quarto', 'basic.qmd', 'control_flow.qmd', 'data_structures.qmd', 'docs', 'exceptions.qmd', 'exceptions_files', 'functions.qmd', 'index.qmd', 'input_and_output.qmd', 'LICENSE', 'modules.qmd', 'more.qmd', 'object_oriented_programming.qmd', 'operators_and_expressions.qmd', 'practical_code', 'problem_solving.qmd', 'python-logo-only.svg', 'README.md', 'README.zh-CN.md', 'references.bib', 'references.qmd', 'standard_library.ipynb', 'standard_library.qmd', 'standard_library_files', '_book', '_quarto.yml']

11.2.3 os.mkdir()

os.mkdir()创建一个新目录。

import os

os.mkdir(r'practical_code/standard_library/newdir')

11.2.4 os.remove()

os.remove()删除一个文件。

import os

os.remove(r'practical_code/standard_library/newdir')

11.2.5 os.rmdir()

os.rmdir()删除一个目录。

import os

os.rmdir(r'practical_code/standard_library/newdir')

11.3 logging模块

logging模块提供了一个灵活的方式来记录日志信息。

logging模块的基本语法如下：

import logging

logging.debug('This is a debug message')
logging.info('This is an info message')
logging.warning('This is a warning message')
logging.error('This is an error message')
logging.critical('This is a critical message')
WARNING:root:This is a warning message
ERROR:root:This is an error message
CRITICAL:root:This is a critical message

此外，logging模块还提供了一些配置选项，可以将日志信息输出到指定位置，或者指定日志的格式、等级等。

以下是一个更复杂一点的例子：

import os
import logging

logging_file = os.path.join(os.getcwd(), 'practical_code/standard_library/logging.log')

print("Logging to", logging_file)

logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    filename=logging_file,
    filemode='w'
)

logging.debug('This is a debug message')
logging.info('This is an info message')
logging.warning('This is a warning message')
WARNING:root:This is a warning message
Logging to C:\Code\MyProjects\My-Python-Learning\practical_code/standard_library/logging.log

在上述的示例代码中：

  • logging_file是日志文件的路径，这里配置为practical_code/standard_library/logging.log
  • logging.basicConfig()函数用于配置日志的格式、等级、输出位置等
  • logging.debug()、logging.info()、logging.warning()等函数用于记录日志信息
  • filemode='w'表示以写入模式打开日志文件，如果文件已经存在，会被覆盖
  • %(asctime)s、%(name)s、%(levelname)s、%(message)s是日志格式的占位符，分别表示日志记录的时间、日志记录的名称、日志的等级、日志的信息
